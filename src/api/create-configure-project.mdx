## Prerequisites

- A Sentry account with an organization-level role of Admin or higher <!-- TODO SHANA confirm -->

We recommend using a free [Sentry developer account](https://sentry.io/pricing/) for this tutorial.

## Create a new project

Create a project with the default issue alert rule turned off

Use the [Create a New Project](https://docs.sentry.io/api/projects/create-a-new-project/) endpoint

- Path Parameters
    - `organization_slug`
    - `team_slug`
- Query Parameters
    - `name`
    - `platform`
    - `default_rules=false`

## Create a new alert for that project

[PLACEHOLDER] Create a New Issue Alert Rule endpoint

- `POST /api/0/{organization_slug}/{project_slug}/rules/`
- In [src/sentry/api/endpoints/project_rules.py](https://github.com/getsentry/sentry/blob/7a826c993a69e0d5a25ceb368483d34de46896c0/src/sentry/api/endpoints/project_rules.py#L49)
