If you use the current version of our Python SDK, distributed tracing works out of the box for the following frameworks:

- <PlatformLink to="/integrations/django/">Django</PlatformLink>
- <PlatformLink to="/integrations/fastapi/">FastAPI</PlatformLink>
- <PlatformLink to="/integrations/flask/">Flask</PlatformLink>
- <PlatformLink to="/integrations/bottle/">Bottle</PlatformLink>
- <PlatformLink to="/integrations/falcon/">Falcon</PlatformLink>
- <PlatformLink to="/integrations/pyramid/">Pyramid</PlatformLink>
- <PlatformLink to="/integrations/quart/">Quart</PlatformLink>
- <PlatformLink to="/integrations/starlette/">Starlette</PlatformLink>
- <PlatformLink to="/integrations/tornado/">Tornado</PlatformLink>

<Note>

If you use version `1.25.x` or below, you need to have our <PlatformLink to="/performance/">performance monitoring</PlatformLink> feature enabled for distributed tracing to work. If you don't want to activate performance monitoring, or you don't use any of the above frameworks, you can also set up <PlatformLink to="/distributed-tracing/custom-instrumentation/">custom instrumentation</PlatformLink>.

</Note>
